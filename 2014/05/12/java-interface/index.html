简介

接口（interface）使Java中“抽象”的概念更深入了一层。它定义了接口实现类的基本形式：

  • 基本数据类型的数据成员，但它们默认都是static和final的；

  • 方法的名字、参数列表以及返回类型，但不能包含其具体实现，默认是public的；

接口只是提供了一种形式，并不提供具体实现上的细节。接口自述：“对于实现我的所有类，看起来都应该像我这个样子”。

创建接口

可以通过interface关键字来创建一个接口。

为了生成一个与特定接口相符的类，需要使用implements（实现）关键字。这里所要表达的意思是：”接口看起来像那个样子，这儿是它具体的工作细节”。

“多重继承”

接口只是比抽象类更“纯”的一种形式。但它的用途远不止这些。在实际的开发中，我们常常需要表达这样一个意思：“x属于a，也属于b，也属于c”。在C++中可以通过其多重继承机制实现。由于Java是单继承机制，所以我们不能通过C++中的方法来实现。这个时候，接口就派上用场了。可以通过将一个类同时实现（implements）多个接口来实现“多重继承”。

扩展接口

利用继承技术，可方便地为一个接口添加新的方法声明，也可以将几个接口合并成一个新接口。在这两种情况下，最终得到的都是一个新接口。

常数分组

由于置入一个接口的所有字段都自动具有 static 和 final 属性，所以接口是对常数值进行分组的一个好工具，它具有与 C 或 C++的 enum 非常相似的效果。
